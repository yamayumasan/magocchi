# 孫っち - コンセプトドキュメント

> 最終更新: 2024年12月
> ステータス: ドラフト

## 1. プロジェクト概要

**孫っち**は、高齢者向けの音声AIアシスタントWebアプリケーションです。

自然な日本語での会話を通じて、日常のさまざまな操作を音声だけで完結できることを目指します。「話しかけるだけで何でもできる」体験を提供し、デジタルデバイドを解消します。

## 2. ビジョン・ミッション

### ビジョン
> すべての高齢者が、テクノロジーの恩恵を自然に受けられる社会を実現する

### ミッション
> 音声という最も自然なインターフェースで、高齢者の日常生活をサポートする

## 3. 解決したい課題

### 社会的背景
- 日本の65歳以上人口は約3,600万人（総人口の29%）
- 一人暮らし高齢者は約670万人、今後も増加傾向
- スマートフォン・タブレット保有率は上昇しているが、使いこなせていない層が多い
- 既存の音声アシスタント（Alexa、Siri）は機能が限定的で、高齢者には使いにくい

### ユーザーの課題
1. **デジタルデバイド**: アプリの操作が複雑で使えない
2. **孤独感**: 話し相手がいない、日常会話の機会が減少
3. **記憶の補助**: 予定や買い物リストを覚えておくのが難しい
4. **情報アクセス**: 知りたいことがあっても調べ方がわからない

## 4. ターゲットユーザー

### プライマリターゲット
**65歳以上の高齢者（本人利用）**

- スマートフォンやタブレットは持っているが、複雑な操作は苦手
- 一人暮らし、または日中一人になることが多い
- 話し相手が欲しい、日常のちょっとしたことを手伝って欲しい

### ペルソナ例

#### 田中 花子さん（78歳・女性）
- 東京郊外の一軒家で一人暮らし
- 夫を5年前に亡くし、子どもは遠方に住んでいる
- タブレットは孫に買ってもらったが、LINEと写真閲覧しか使えない
- 買い物リストをメモするのが面倒、予定もよく忘れる
- 電話を聞いてくれる相手がいない

#### 佐藤 一郎さん（72歳・男性）
- 妻と二人暮らし、日中は妻がパートで不在
- 定年退職後、社会との接点が減った
- ニュースや天気を知りたいが、スマホの操作が億劫
- 通院の予定を忘れがち

### セカンダリターゲット（将来展開）
- **介護施設**: 入居者向けの会話・見守りツールとして導入
- **自治体**: 高齢者見守り事業の一環として配布
- **高齢者の家族**: 親へのプレゼント、見守り目的での導入

## 5. コアバリュー

### 「話すだけで、なんでもできる」

既存のアプリは画面操作が前提だが、孫っちは**音声がプライマリインターフェース**です。

| 従来のアプリ | 孫っち |
|-------------|--------|
| 画面を見て、ボタンを探して、タップ | 「買い物リストに牛乳を追加して」 |
| カレンダーアプリを開いて、日付を選んで、予定を入力 | 「来週の火曜日に眼科の予約を入れて」 |
| ブラウザを開いて、検索して、読む | 「明日の天気を教えて」 |
| メモアプリを開いて、入力して、保存 | 「孫の太郎は来月誕生日だから覚えておいて」 |

### 差別化ポイント

1. **自然言語での操作**: 定型コマンドではなく、自然な会話で操作
2. **長期記憶**: ユーザーの情報を覚えて、文脈を踏まえた応答
3. **多機能統合**: タスク管理、リマインダー、雑談、情報検索を一つのインターフェースで
4. **高齢者特化UI**: 大きな文字、シンプルな画面、音声中心の操作
5. **親しみやすいキャラクター**: 「孫」のような存在として寄り添う

## 6. MVP機能スコープ（3ヶ月目標）

### 必須機能（Must Have）

#### 6.1 音声会話基盤
- 音声入力（Speech-to-Text）
- 音声出力（Text-to-Speech）
- 自然な日本語での対話（LLM）
- 会話のストリーミング表示

#### 6.2 長期記憶システム
- ユーザープロファイル保存（名前、家族構成、好みなど）
- 会話履歴の保存と検索
- 文脈を踏まえた応答生成

#### 6.3 タスク・リマインダー管理
- 買い物リストの追加・確認・削除
- やることリストの管理
- リマインダーの設定と通知
- 予定の登録と確認

#### 6.4 情報提供
- 天気情報の取得と読み上げ
- 日付・時刻の案内
- 簡単な質問への回答

#### 6.5 基本UI
- 大きなボタンでの会話開始
- 会話履歴の表示
- シンプルな設定画面
- 音声選択機能（MVP: 1種類）

### 将来機能（Future）

- 家族向け見守りダッシュボード
- 複数の声の選択（孫世代・子世代の男女4種類）
- 終活支援（遺書、伝えたいことの記録）
- 健康管理（服薬記録、体調メモ）
- 施設向け管理機能
- 外部サービス連携（LINE通知など）

## 7. 技術的チャレンジ

このプロジェクトで習得・実践する技術領域：

### フロントエンド
- Vue 3 + TypeScript + Composition API
- 音声入力/出力のWeb API統合
- リアルタイムストリーミング表示
- PWA対応（オフライン、プッシュ通知）
- 高齢者向けアクセシビリティ設計

### バックエンド
- Node.js + Express/Fastify + TypeScript
- RESTful API設計
- WebSocket/SSEによるリアルタイム通信
- 認証・認可システム

### AI/ML
- LLM API統合（Claude/GPT）
- 音声認識API（Whisper）
- 音声合成API（ElevenLabs/VOICEVOX）
- Vector Database（Pinecone/Supabase）によるRAG実装
- プロンプトエンジニアリング

### データベース
- PostgreSQL（構造化データ）
- Vector DB（会話履歴、長期記憶）
- データモデリング

### インフラ
- クラウドデプロイ（Vercel/Railway）
- 環境変数管理
- CI/CD

## 8. 成功指標（MVP）

### 定量指標
- 1回の会話セッションで3つ以上のタスクを完了できる
- 音声認識精度: 90%以上（標準的な発話）
- 応答時間: 3秒以内（音声出力開始まで）
- 長期記憶の正確な呼び出し: 95%以上

### 定性指標
- 高齢者（または高齢者を想定したテスター）が直感的に使える
- 「また使いたい」と思える体験
- ポートフォリオとして技術力をアピールできる完成度

## 9. 競合・類似サービスとの比較

| サービス | 特徴 | 孫っちとの違い |
|---------|------|---------------|
| Alexa/Google Home | 音声アシスタント | 定型コマンド中心、長期記憶なし |
| Cotomo | 音声会話AI | 雑談特化、タスク管理機能なし |
| ElliQ | 高齢者向けAI | 専用ハードウェア必要、日本未展開 |
| Replika | AIコンパニオン | 高齢者向け設計ではない |
| 孫っち | 高齢者×音声×多機能 | 音声で何でもできる、長期記憶、日本語特化 |

## 10. リスクと対策

| リスク | 影響 | 対策 |
|-------|------|------|
| 音声認識精度（高齢者の発話） | 高 | 複数のSTTサービス比較、プロンプトでの補正 |
| 音声合成の自然さ | 中 | ElevenLabs等の高品質サービス活用 |
| LLM応答の安全性 | 高 | 適切なシステムプロンプト、フィルタリング |
| API利用コスト | 中 | 効率的なAPI呼び出し設計、キャッシング |
| 3ヶ月での完成 | 中 | スコープの厳格な管理、優先度付け |
