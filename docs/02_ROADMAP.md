# 孫っち - 開発ロードマップ

> 最終更新: 2024年12月

## サービス成功のための全体ステップ

```
[Phase 0: PoC] → [Phase 1: MVP] → [Phase 2: β版] → [Phase 3: 正式版]
    検証           実装            改善             拡張
```

---

## Phase 0: PoC（技術検証）

### 目的
本当に作れるか？技術的に成立するか？を最小限の実装で検証する。

### 検証項目

#### PoC-1: 音声入出力の基本動作
- [ ] ブラウザでの音声録音・送信
- [ ] Whisper APIでの音声認識精度確認
- [ ] TTS（ElevenLabs/VOICEVOX）での音声出力確認
- **検証基準**: 標準的な発話で認識精度80%以上

#### PoC-2: LLM会話の成立
- [ ] Claude/GPT APIとの接続
- [ ] システムプロンプトによるペルソナ設定
- [ ] ストリーミングレスポンスの実装
- **検証基準**: 自然な会話が成立すること

#### PoC-3: 長期記憶の実現可能性
- [ ] Vector DB（Supabase/Pinecone）の動作確認
- [ ] 会話履歴の埋め込み・検索
- [ ] 関連情報の呼び出し精度確認
- **検証基準**: 過去の会話を適切に参照できること

#### PoC-4: エンドツーエンド統合
- [ ] 音声入力→LLM→音声出力の一連の流れ
- [ ] 応答時間の計測（目標: 3秒以内）
- **検証基準**: 一連の流れが途切れなく動作すること

### 成果物
- 技術検証レポート
- Go/No-Go判断

## Phase 1: MVP（最小限の製品）

### 目的
ユーザーに価値を提供できる最小限の機能を実装する。

### スコープ
→ 詳細は [MVP仕様](./03_MVP_SPEC.md) を参照

### 機能
1. 音声での会話（入力・出力）
2. 買い物リスト管理
3. リマインダー設定
4. 基本的な質問応答（天気、日時など）
5. シンプルなUI（大きなボタン、見やすい表示）

### 成功基準
- 1セッションで3つ以上のタスクを完了できる
- 高齢者（想定テスター）が説明なしで基本操作できる
- 応答時間3秒以内

---

## Phase 2: β版（改善・拡張）

### 目的
実際のユーザーフィードバックを元に改善する。

### 追加機能
- ユーザー認証・個人データ保存
- 複数の声の選択
- より高度な長期記憶
- プッシュ通知によるリマインダー
- UI/UXの改善

### 検証
- 実際の高齢者（親族など）でのテスト
- 継続利用率の計測
- フィードバック収集

---

## Phase 3: 正式版・拡張

### 目的

### 機能拡張

- 施設向け管理機能

### ビジネス展開

- マーケティング
- パートナーシップ（介護施設、自治体）

## 開発マイルストーン（詳細）

### Month 1: 基盤構築（PoC + MVP基盤）

**Week 1-2: PoC実施**

## 判断ポイント

### PoC後のGo/No-Go
以下のいずれかに該当する場合、アプローチを再検討:
- 音声認識精度が70%未満
- 応答時間が5秒を超える
- API コストが想定の3倍以上

### MVP後の判断
以下を確認して次フェーズへ:
- ターゲットユーザーが価値を感じるか
- 技術的に拡張可能か
- 継続して開発するモチベーションがあるか
